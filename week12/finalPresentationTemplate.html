Presentation Template (15-25 min)

Overview

Total Duration: 15-25 minutes

  • Presentation: 12-20 minutes
  • Q&A Session: 3-5 minutes

Objective: Demonstrate chatbot development, functionality, compliance, and Agile methodology application

Section Breakdown

1. Introduction (1-2 minutes)

  • Greeting and team introduction
  • Project overview: Chatbot purpose (e.g., handling course registration, campus resources, academic advising)
  • Agenda outline and transition to main content

2. Chatbot Functionality and Design - UI (2-3 minutes)

Key Points:

  • Intuitive design
  • Responsiveness
  • Integration with Scrum/Agile (iterative development, user stories)

Demo: Live UI walkthrough

Evidence: Frontend framework choices (e.g., React.js) and sprint enhancements

3. Chatbot Functionality and Design - Agent Design (3-4 minutes)

Key Points:

  • Core logic
  • Data preparation
  • Advanced features (e.g., RAG, multi-turn conversations, API integrations)
  • Alignment with project needs and organizational use case

Evidence: Test results, accuracy metrics, charts

4. Performance Evaluation and Monitoring (2-3 minutes)

Key Points:

  • Custom metrics (e.g., response accuracy, user satisfaction)
  • Monitoring tools
  • User feedback collection and iteration

Evidence: Feedback logs, analysis charts

5. Agile/Scrum Application and Planning (2-3 minutes)

Key Points:

  • Use of sprints, backlogs, artifacts
  • Team participation
  • How Agile drove project success

Evidence: Scrum artifacts (e.g., screen captures of backlogs, iteration histories)

6. Legal and Ethical Considerations (1-2 minutes)

Key Points:

  • Compliance with Hong Kong data privacy and AI regulations
  • Implementation of data protection principles
  • Ethical AI deployment practices

Evidence: Reference to official guidelines and frameworks

7. Documentation (1-2 minutes)

Key Points:

  • Scrum artifacts, code annotations, feedback logs, iteration histories
  • How documentation supports project understanding and maintainability
  • Importance of clear documentation for future development and onboarding

Evidence: Slides with examples or references to submitted materials

8. Conclusion (1 minute)

Key Points:

  • Summary of key achievements and alignment with organizational needs
  • Overall impact and future potential
  • Thank the audience
  • Transition to Q&A

9. Q&A Section (3-5 minutes)

Participants: All team members

Format:

  • Open floor for questions
  • All team members available to respond, ensuring balanced participation
  • Address queries on features, process, evaluation, etc.

Presentation Tips

  • Time Management: Stick to allocated times for each section (total 15-25 minutes
  • Visual Aids: Use slides, diagrams, and live demos effectively
  • Engagement: Maintain eye contact and address the audience directly
  • Preparation: Rehearse multiple times as a team to ensure flow
  • Evidence: Always show supporting evidence (screenshots, charts, logs)
  • Criterion Alignment: Clearly map each section to the rubric criteria
